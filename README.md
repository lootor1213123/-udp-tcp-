# 使用udp模拟可靠tcp
使用py在应用层实现了udp的可靠传输（学习教学项目）

项目还有很多不太完善的地方，比如tcp的滑动窗口算法

流量控制和阻塞控制还有超时重传没有完善，所以本质还是一个参考tcp可靠性的最小化可执行udp

简易udp缺点：

每一次发送方发，接收方需要返回确认的ack给对方，所以导致后面的包都在等对方，这也是停等协议的一大缺点，后续改进的话，就是利用滑动窗口机制和多线程，一次性多发送几个包过去，然后接收方看seq，接收方随后返回他的seq和ack，发送方接着看回复方的ack是否 ＝ seq + 1 即可（此处的seq为发送端发的包的序号）
 
在此发表一下我的看法：

UDP是最小化可执行TCP
本质都是从A到B发送文件

我们协议本身只是对数据荷载前面的报头进行不同的处理
